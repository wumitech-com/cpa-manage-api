# K8s 端口转发完整指南

## 网络架构
```
本地 Windows
  ↓
第一层跳板机: 106.75.152.136 (ubuntu)
  ↓
第二层跳板机: 103.149.27.5 (ubuntu) ← 这里运行 kubectl port-forward
  ↓
K8s Pod: cpa-manage-api-5954dbc5b5-kjhld (端口 8081)
```

## 需要确认的信息

### 1. 密钥配置情况
- [ ] 本地到第一层跳板机（106.75.152.136）：有密钥 ✓
- [ ] 第一层到第二层跳板机（103.149.27.5）：需要确认
- [ ] 本地到第二层跳板机：没有密钥 ✗

### 2. 当前状态
- [ ] kubectl port-forward 是否在第二层跳板机上运行？
- [ ] 第一层跳板机上是否可以 SSH 到第二层跳板机？
- [ ] 第一层跳板机到第二层跳板机是否需要密码？

## 方案选择

### 方案 A：分步建立隧道（推荐）
需要：
1. 第一层跳板机可以 SSH 到第二层跳板机（有密钥或密码）
2. 在第二层跳板机上运行 kubectl port-forward

### 方案 B：使用 K8s 公网地址
如果服务已通过 NodePort/LoadBalancer 暴露，可以直接访问

### 方案 C：修改 kubectl port-forward 监听公网
需要修改 kubectl port-forward 监听 0.0.0.0
