# 关于进程数量的说明

## 为什么有两个进程？

当使用以下命令启动应用时：
```bash
bash -c "cd /home/ubuntu/cpa-manage-api && nohup java -jar cpa-manage-api-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod > app.log 2>&1 &"
```

会看到两个进程：
1. **bash 进程** (PID 65383) - 执行命令的 shell
2. **java 进程** (PID 65384) - 实际运行的应用程序

这是**正常现象**，因为：
- `bash -c` 会创建一个 bash 子进程来执行命令
- `nohup` 和 `&` 会让 java 进程在后台运行
- bash 进程会等待 java 进程启动后退出（或保持运行）

## 如何只看到一个进程？

### 方法1：直接使用 nohup（推荐）
```bash
cd /home/ubuntu/cpa-manage-api
nohup java -jar cpa-manage-api-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod > app.log 2>&1 &
```

这样只会有一个 java 进程。

### 方法2：使用 systemd 服务（生产环境推荐）
创建 `/etc/systemd/system/cpa-manage-api.service`：
```ini
[Unit]
Description=CPA Manage API
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/cpa-manage-api
ExecStart=/usr/bin/java -jar /home/ubuntu/cpa-manage-api/cpa-manage-api-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
Restart=always
StandardOutput=append:/home/ubuntu/cpa-manage-api/logs/cpa-manage-api.log
StandardError=append:/home/ubuntu/cpa-manage-api/logs/cpa-manage-api-error.log

[Install]
WantedBy=multi-user.target
```

然后：
```bash
sudo systemctl daemon-reload
sudo systemctl enable cpa-manage-api
sudo systemctl start cpa-manage-api
```

## 检查应用是否正常运行

### 查看进程
```bash
ps aux | grep cpa-manage-api | grep -v grep
```

应该只看到一个 java 进程。

### 查看端口
```bash
netstat -tlnp | grep 8081
# 或
ss -tlnp | grep 8081
```

### 查看日志
```bash
tail -f /home/ubuntu/cpa-manage-api/logs/cpa-manage-api.log
```

## 当前情况

您当前看到的两个进程是正常的：
- bash 进程会很快退出（或保持运行但不占用资源）
- java 进程是实际的应用，占用 CPU 和内存

如果担心资源占用，可以：
1. 等待几分钟，bash 进程可能会自动退出
2. 或者使用上面的方法1直接启动，避免 bash -c
