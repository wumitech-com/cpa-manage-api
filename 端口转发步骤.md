# K8s 端口转发完整步骤

## 前提条件
- ✅ kubectl port-forward 已在第二层跳板机运行
- ✅ 第一层跳板机可以 SSH 到第二层跳板机

## 执行步骤

### 步骤 1：在第一层跳板机（106.75.152.136）上执行

SSH 连接到第一层跳板机，然后运行：

```bash
ssh -N -L 5556:127.0.0.1:5556 ubuntu@103.149.27.5
```

**说明**：
- 这个命令会一直运行，保持窗口打开
- 它会在第一层跳板机上监听 `127.0.0.1:5556`
- 并转发到第二层跳板机的 `127.0.0.1:5556`

### 步骤 2：在本地 Windows PowerShell 执行

```powershell
ssh -N -L 5556:127.0.0.1:5556 -o StrictHostKeyChecking=no -i ~/.ssh/id_rsa ubuntu@106.75.152.136
```

**或者使用脚本**：
```powershell
.\port-forward.ps1
```

### 步骤 3：测试连接

在浏览器访问或使用 curl：
```powershell
curl http://localhost:5556
```

浏览器访问：`http://localhost:5556`

## 完整连接链路

```
本地 Windows (localhost:5556)
  ↓ SSH 隧道（步骤2）
第一层跳板机 106.75.152.136 (127.0.0.1:5556)
  ↓ SSH 隧道（步骤1）
第二层跳板机 103.149.27.5 (127.0.0.1:5556)
  ↓ kubectl port-forward（已运行）
K8s Pod cpa-manage-api-5954dbc5b5-kjhld (8081)
```

## 注意事项

1. **步骤 1 和步骤 2 都需要保持运行**，不要关闭终端窗口
2. 如果连接失败，检查：
   - 第一层跳板机上的 SSH 隧道是否正常运行
   - kubectl port-forward 是否在第二层跳板机上运行
   - 本地 SSH 隧道是否正常建立

## 停止端口转发

- 在对应的终端窗口按 `Ctrl+C` 停止
